Editor

The editor represents one instance of SirTrevor. It is always tied to a <textarea> that represents the data from your database.

initialize

new SirTrevor.Editor([ attributes ])

Creating a new instance of the editor will invoke the initialize function. The only required element within the attributes object is the el which must refer to a jQuery object containing a textarea.

build

editor.build()

Build will loop through the data of the Editor and either create the necessary blocks from the data, or create a default block if no data is present.

store

editor.store(method, editor, [options])

Store is just a wrapper for our EditorStore. Any function for data storage will call this method.

createBlock

editor.createBlock(type, [data])

Invoking the createBlock function will try and create a block of the type specified using the type parameter. Passing the optional data parameter will create the block with the data specified.

When creating a block we consider the following:

  • Whether there is an instance limit for this block (passed using the blockTypeLimits option on init)
  • Whether there is a global limit for this block
  • Whether there is a total block limit for the editor (passed using the blockLimit option on init)

removeBlock

editor.removeBlock(SirTrevor.Block)

Calling removeBlock will remove the current block passed from the editors blocks array, and also remove it from the dom.
